Always follow this workflow:

1. Read documentation.
2. Understand context.
3. Explain the implementation plan.
4. Wait for approval before implementing.
5. Implement.
6. Perform a Self Review.
7. Perform a Documentation Review.
8. Perform an ADR Review.
9. Perform a CHANGELOG Review.
10. Suggest a Conventional Commit.

Never skip planning.

Never implement multiple unrelated features in the same iteration.

Prefer small, reviewable changes.

Documentation is part of the implementation, not an optional follow-up task.
