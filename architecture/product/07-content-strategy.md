# Content Strategy

> This document defines how content should be written, organized and presented throughout the experience.
>
> The goal is not to maximize information.
>
> The goal is to maximize meaning.

---

# Purpose

The website should never feel like a repository of videos.

It should feel like a collection of stories.

Every piece of content must answer one question:

> Does this help visitors understand Paulera's creative vision?

If the answer is no, it probably doesn't belong on the homepage.

---

# Content Philosophy

The content hierarchy follows storytelling instead of chronology.

Visitors should understand the artist before the work.

The work before the technical details.

The experience before the specifications.

---

# Homepage Content

The homepage should never try to explain everything.

Its role is to invite exploration.

Each section should reveal only enough information to encourage scrolling.

---

## Roots

Purpose

Present identity.

Content

- Manifesto
- Philosophy
- Inspiration
- Short biography
- Portrait

Avoid

- Long biography
- CV
- Lists of achievements

---

## Trunk

Purpose

Build trust.

Content

- Timeline
- Milestones
- Selected clients
- Awards
- Testimonials

Everything should reinforce credibility without sounding promotional.

---

## Branches

Purpose

Present creative areas.

Each branch contains:

- Title
- Short description
- Featured projects

Never overload categories.

---

## Fruits

Purpose

Present stories.

Each project receives equal narrative respect.

Featured projects appear first.

Older works remain accessible.

---

## Seeds

Purpose

Invite collaboration.

Keep this section simple.

Human.

Optimistic.

Warm.

---

# Case Structure

Every Case should follow the same storytelling rhythm.

---

## Hero

Large visual impact.

Minimal text.

Immediately establish atmosphere.

---

## Context

Why did this project exist?

Who was the client?

What problem needed solving?

---

## Creative Challenge

What made this project interesting?

What constraints existed?

What decisions were taken?

---

## Process

Optional.

Only when relevant.

Behind-the-scenes.

References.

Creative sketches.

Production moments.

---

## Final Result

Embedded video.

Gallery.

Highlights.

---

## Reflection

What did Paulera learn?

What made this project memorable?

This section humanizes the portfolio.

---

## Credits

Always recognize collaborators.

This reinforces professionalism.

---

## Related Stories

Suggest projects with similar themes or techniques.

---

# Writing Style

The copy should sound like a filmmaker.

Not an advertiser.

Sentences should be concise.

Natural.

Honest.

Avoid corporate language.

Avoid exaggeration.

---

# Headlines

Prefer curiosity over explanation.

Examples

Instead of

"About Me"

Use

"Where every story begins."

---

Instead of

"Our Projects"

Use

"Stories that became reality."

---

Instead of

"Contact"

Use

"Let's plant the next seed."

---

# Calls to Action

CTAs should invite collaboration.

Never pressure.

Examples

"Vamos criar a próxima história?"

"Pronto para tirar uma ideia do papel?"

"Qual história você quer contar?"

Avoid

"Solicite um orçamento."

"Entre em contato."

"Fale conosco."

---

# Video Guidelines

Videos are the heart of the experience.

They should never compete with each other.

One hero.

One focus.

Minimal controls.

Fast loading.

High quality.

---

# Image Guidelines

Images should support the story.

Not decorate the layout.

Every image should have a purpose.

If an image can be removed without affecting understanding,

it probably should.

---

# Testimonials

Only include testimonials that communicate trust.

Prefer short quotes.

Real names.

Real companies.

Authentic language.

---

# Client Logos

Display selectively.

Quality over quantity.

Logos should reinforce credibility,

not dominate the interface.

---

# SEO Content

Every Case should include:

- Title
- Description
- Keywords
- Structured Data
- Open Graph Image
- Alt Text
- Canonical URL

The content should remain human-first.

Never write for search engines.

---

# Content Maintenance

Adding a new project should require only content creation.

No layout changes.

No design changes.

No development changes.

The CMS should support this naturally.

---

# Future Content

The architecture supports future additions:

- Creative Journal
- Production Notes
- Behind the Scenes
- Director's Commentary
- Awards
- Speaking Engagements

Each new content type should inherit the same storytelling principles.

---

# Decision Log

## D-019

Every project becomes a story.

Reason:

Stories are memorable.

Lists are not.

---

## D-020

Homepage content introduces.

Case pages explain.

Reason:

Avoid overwhelming first-time visitors.

---

## D-021

Copy prioritizes authenticity over marketing.

Reason:

People hire artists because of their vision.

Not because of slogans.

# AI Context

Never generate generic marketing copy.

Write like a filmmaker reflecting on the creative process.

Every paragraph should reinforce authenticity.

If a sentence sounds like advertising, rewrite it.

Content should feel personal, cinematic and timeless.
